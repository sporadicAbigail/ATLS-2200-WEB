Who is web accessibility for?

Everyone

Things to code for

Auditory

People who are hard of hearing and require captions/transcript. like captions on videos (a youtube video)

Cognitie

Difficulty focusing. deliver content in more than one way, like a blog has text + images + video

don't have an entire block of text

neurological

avoid flashing lights

physical

not everyone will use a mouse or keyboard or a stick, so make it as easy as possible.

speech

some websites require you to interact with a website verbally, but try not to do that

Visual

people who are blind or colorblind or nearsighted.

they use screenreader

We are coding for People

Web content accessibility guidelines

perceivable - perceive it through another sense

operable - operate with both a mouse and a keyboard

understandable - understood by a range of people

robust - works across multiple platforms, and it needs to continue to work into the futre as technology evolves

Website grades

A: some issues must be resolved and represents significant barriers

AA: generally accepted

AAA: very few websites are this level, some issues could be resolved

Check the color contrast - are the color contrast good?

Code with accessibility in mind

webpage title

website semantic: headings structure, navigation. We know section, H1-H6, we need headers in each section. Navigation can have a navbar with list. as long as you have semantics, it should be easier to navigate. test if your website would work the same with or without css. Would you understand it the same?

Links: descriptive -> 'Read more' vs 'Climate Change Full Story'

Alt test: image description, if the image was removed, would the user still get all the information from the page? The only time when your alt text would be empty is if it were decorative, like a logo. It should be neither the title of your image nor an essay. Enough that you know what the photo is above.

WAI-ARIA

Accessible rich internet applications

Helps developers create widgets, applications, and web interactive in general that meet WCAG recommendations

it's just super helpful for accessibility

aria attributes to add to make anything you want accessible

Design with accessibility in mind

Color contrast so it's easy to read at any font size. Contrast is different between 40px and 12px

Text wrapping: make sure you don't have to scroll horizontally and vertically to read text. It's annoying to have to scroll back and forth horizontally. If you can make it so your text wraps, that helps everyone. Especially for slow readers as well. Responsive design helps with that.
